# @soundweave/ui

Shared UI components for Soundweave applications.

## What It Owns

- Reusable UI primitives shared between Studio and Docs
- Design system components (buttons, inputs, tabs, badges, etc.)

## Status

Placeholder package. UI components currently live inline in Studio screen files. This package will be extracted and unified during the visual polish pass (Phase 22).

## What It Does Not Own

- Application state management (see Studio store)
- Screen layout and routing
- Business logic
